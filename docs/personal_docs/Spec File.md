## Goal
The system need to be able to ingest large quentities of data and embed it so that users would be able to query it using natural language.

## Constraints 
The indexing-embeding phase has to:
1. Allow for very large files
2. Be reasonably quick, exact time frame not yet known
3. pull data from **different** sources
4. be continuous, if new files are created, they are automatically embbeded and indexed
5. MUST support code searching and keywords very accuratly.
6. Performance and usage monitoring - Bonus

## Services
From a user's POV, the system only has one service. After setting up the data collectors, or adapters, the user can use a CLI to query his/her own data using natural language and get a natural language response.
